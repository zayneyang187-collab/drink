
1. 已完成的功能（当前代码状态）

已完成小程序四大 Tab 架构：选酒 / 榜单 / 点评 / 我的，页面中文化和 iOS Grouped 风格基础样式已落地。
首页已重构：支持场景/预算/人群/偏好推荐入口、本周热门预览、点评入口；并修复了标题安全区与快速推荐区过高问题。
推荐与详情链路可跑通：推荐列表 -> 详情页 -> 评分页。
详情页已实现：场景切换、分数展示、情绪化文案、稳定性提示、推荐理由、复制话术、分享、收藏。
点评页已实现：搜索选酒、最近点评；搜索无结果时可“新建酒款并去点评”。
后端主要接口已具备：recommend/rank/search/detail/ratings/me/favorites/reports，并新增 POST /api/liquors（创建酒款）。
评分算法服务（贝叶斯加权、权重、理由生成等）和 SQLite 数据读写链路可用。
2. 使用的技术栈和约定

前端：微信小程序原生 MINA（WXML/WXSS/JS），不使用 TS。
后端：Node.js + Express，全部 CommonJS（module.exports）。
数据库：SQLite（better-sqlite3），本地文件运行。
API 约定：前缀 /api，默认端口 3000，CORS 已开启。
返回约定：成功返回对象/数组；失败返回 { ok:false, error:{ code, message } }。
小程序请求封装：request.js（baseUrl、超时、错误提示）。
